## 3. 機能詳細

### FR-2 チャット & エージェント

#### 3.1 入力 → 検索プラン提示（実行前確認）
1. ユーザーが自然言語でICPを入力
   - 例: 「フロリダで技術者を採用中、かつGoogle広告を出しているHVAC企業」
2. エージェント（LLM）が**構造化ICP**に変換して提示:
   ```
   ┌ 検索プラン ──────────────────────┐
   │ 業種: HVAC（空調設備）             │
   │ 地域: 米国フロリダ州               │
   │ シグナル: 技術者を採用中 / Google広告出稿中 │
   │ 検索ソース: 地図, 求人, 広告透明性, 企業サイト │
   │ 想定取得件数: 〜200件              │
   │ 見積りクレジット: 約120           │
   │           [修正する] [実行する]    │
   └──────────────────────────────┘
   ```
3. ユーザーが `[実行する]` を押すと非同期ジョブ開始。
4. **この検索プラン（構造化ICP＋ソース＋パラメータ）を `search_plan` としてスナップショット保存。** → 後日そのまま再実行して同じ条件を再現できる（FR-2.5 差別化）。

#### 3.2 進捗ストリーミング（FR-2.3）
実行中、チャットに進捗を逐次表示:
```
🔍 地図ソースを検索中… 84件の候補
🔍 求人シグナルを照合中… 31件が該当
✅ メール検証中… 22件が検証成功
✅ 完了: 22件のリードを取得（消費 118クレジット）
```
右ペインの結果テーブルにも行が随時追加される。

#### 3.3 追加指示による絞り込み（FR-2.4）
- 「従業員10名以上に絞って」「メールがある会社だけ」等の追加発話で、既存結果に対しフィルタ/再検索を実行。会話履歴はコンテキストとして保持。

---

### FR-3 データソース検索

#### 3.4 コネクタ層の挙動
- エージェントが検索プランから「どのコネクタをどのパラメータで叩くか」を決定（ツール呼び出し）。
- 各コネクタは統一インターフェース `search(query): LeadCandidate[]` を実装。
- 市場設定（`market: "JP" | "GLOBAL"`）でコネクタ群を切替。
- 詳細な内部設計は [04_システム設計書](./04_システム設計書.md) の「エージェント & コネクタ」節。

#### 3.5 名寄せ・重複排除（FR-3.3）
- 会社名・ドメイン・住所・電話をキーに正規化 → 重複マージ。
- 各リードに `sources[]`（見つかった全ソースと元URL）を保持し、出典を明示。

---

### FR-4 検証・エンリッチメント

#### 3.6 多段ウォーターフォール検証（FR-4.1〜4.4）
- メール: 複数プロバイダを順に試行（syntax → MX → SMTP/プロバイダA → B …）。最初に確度が閾値を超えた時点で確定。
- 各項目に**信頼度スコア（0-100）**を付与し、テーブルでバッジ表示（緑/黄/赤）。
- **マッチ成功時のみクレジット消費**（FR-4.5）。失敗は課金しない。
- 詳細アルゴリズムは [08_精度向上戦略](./08_精度向上戦略.md)。

#### 3.7 リード詳細ドロワー
行クリックで右からドロワー展開:
```
株式会社サンプル歯科
├ ドメイン: sample-dc.jp
├ メール: info@sample-dc.jp  [信頼度 92 ✅]
├ 電話: 03-xxxx-xxxx        [信頼度 78 🟡]
├ 出典: Googleマップ, 求人サイト, 企業サイト
├ シグナル: 歯科衛生士を募集中（2026-06）
├ エンリッチ: 従業員 5-10 / 予約システム導入
└ [CRMに送る] [除外] [お気に入り]
```

---

### FR-5 リード管理

- **テーブル**: ソート/列フィルタ/検索。列は表示切替可能。
- **保存リスト**: 現在の結果を命名して保存。後から追記も可。
- **エクスポート**: CSV（無料）。列選択可。
- **ステータス**: 各行に `new / favorite / excluded` を付与。

---

### FR-6 クレジット & 課金（決済は最後に実装）

- ヘッダに残高常時表示。
- 消費が伴う操作の前に**見積りを提示**（例: 「この実行で約120消費、残高1,840」）。
- 消費履歴ページ: 日時・操作・消費量・関連ジョブ。
- プラン別に月次付与・リセット。
- Stripe連携UI（プラン変更・追加購入）は骨組みのみ用意し、**キー投入は最終フェーズ**。

---

### FR-7 公開API & CRM連携（差別化）

- **APIキー管理画面**: 発行・失効・スコープ。
- **REST API**: `POST /v1/search`（検索ジョブ作成）、`GET /v1/jobs/:id`（結果取得）。詳細は [06](./06_外部API連携設計.md)。
- **CRM書き出し**: リード選択 →「HubSpotに送る/Salesforceに送る」。
- **Webhook**: ジョブ完了時に通知。

## 4. 状態遷移（検索ジョブ）

```
draft(プラン提示) ──実行──▶ queued ──▶ running ──▶ verifying ──▶ done
                                          │                       ▲
                                          └── partial(部分完了) ───┘
                          （失敗時）running ──▶ failed（部分結果は保持）
```

## 5. エラー/エッジケース

| ケース | 挙動 |
|--------|------|
| 外部ソースが一部失敗 | 部分結果を返し、失敗ソースを明示。課金は成功分のみ |
| クレジット不足 | 実行前にブロックし、見積り差分とアップグレード導線を提示 |
| 0件ヒット | プラン緩和の提案（地域拡大・シグナル削除等）を提示 |
| 同一プロンプト再実行 | スナップショットから再現 or 差分検索を選択可 |
| MOCK_MODE | 決定的なダミーデータを返し全画面が動作 |
