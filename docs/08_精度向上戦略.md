# 08. 精度向上戦略（origami超えのコア）

ご指示「既存機能を揃えるだけでなく、**精度を高める**」への回答。
origami の弱点＝①結果のブレ（再現性）②検証幅の狭さ③出典の不透明さ、をここで潰す。

---

## 1. 精度を構成する3要素

| 要素 | 定義 | 打ち手 |
|------|------|--------|
| **Recall（網羅性）** | 該当企業をどれだけ取りこぼさないか | マルチソース横断＋名寄せ |
| **Precision（的確性）** | 出した結果がどれだけ本当に該当するか | ICP適合スコアリング |
| **Validity（有効性）** | 連絡先が実際に到達可能か | 多段ウォーターフォール検証 |
| **Reproducibility（再現性）** | 同条件で同結果を返せるか | プランのスナップショット |

---

## 2. Recall：取りこぼしを減らす

- **マルチソース横断**: 1ソースが漏らしても他ソースで拾う（地図＋求人＋企業DB＋SNS）。
- **名寄せ（Entity Resolution）**: 会社名・ドメイン・電話・正規化住所をキーに重複統合。曖昧一致は pgvector の埋め込み類似で補助。
- **クエリ拡張**: LLMで業種の同義語・関連キーワードを展開（例: 「歯科」→「デンタルクリニック/歯科医院/矯正歯科」）。

---

## 3. Precision：的外れを減らす（ICP適合スコアリング）

各リード候補に対し、検索プランの構造化ICPとの適合度を採点。

```
適合スコア = Σ( 条件一致 × 重み )
  例: 業種一致(0.4) + 地域一致(0.3) + シグナル一致(0.3)
```
- シグナル（採用中/資金調達/広告出稿）は**根拠URLとセット**で保持（`lead_sources`）。
- 閾値未満はデフォルト非表示（ユーザーが緩められる）。

---

## 4. Validity：連絡先の有効性（多段ウォーターフォール検証）

origami（メール5/電話9）より**段数と根拠の透明性**で上回る。

### メール検証パイプライン
```
1) Syntax チェック（形式）
2) ドメイン/MXレコード存在
3) キャッチオール判定
4) プロバイダA（到達性/リスク）
5) プロバイダB（別ロジックで裏取り）
   … 確度が閾値超で確定・以降スキップ（コスト最適化）
→ score(0-100) と result(valid/risky/invalid/unknown) を付与
```

### 電話検証パイプライン
```
1) E.164 正規化（国別: JP=+81 / US=+1 …）
2) 番号帯・キャリア有効性
3) 固定/携帯/IP 判定
→ score付与
```

### 設計上の要点
- **成功時のみ課金**（失敗は無料）→ 信頼と粗利の両立。
- 各段の結果を `lead_verifications` に記録 → **なぜその判定か**を説明可能。
- 同一ドメイン/番号はキャッシュして重複コスト削減。

---

## 5. Reproducibility：再現性（最大の差別化）

origami の「同じプロンプトで結果がブレる」を構造で解決。

- 実行前に **構造化ICP＋コネクタ＋パラメータ** を `search_plans` に凍結保存。
- 再実行時は2モードを提供:
  - **再現モード**: 保存プランでそのまま再取得（条件を固定）
  - **差分モード**: 前回以降の新規/更新のみ取得（鮮度更新）
- LLMの非決定性は「プランニング段階で確定 → 実行は決定的」に分離することで抑制（temperature低め＋スキーマ固定＋プラン人手承認）。

---

## 6. 信頼度スコアの可視化（UX）

- 各連絡先にバッジ: 🟢 高(80-100) / 🟡 中(50-79) / 🔴 低(0-49)。
- 総合 `confidence` はフィールドスコアの加重平均。
- ドロワーで**根拠（出典URL・検証プロバイダ・判定）**を開示 → ユーザーの信頼を獲得。

---

## 7. 継続的な精度改善ループ

- **フィードバック取り込み**: ユーザーの「除外/お気に入り」や bounce 報告を教師信号に。
- **カバレッジ計測**: 業界×地域ごとにヒット率・検証成功率を定点観測。
- **プロバイダ入替**: アダプタ層でA/B比較し、精度/単価が良い方へ差替え。
- **原価×精度の最適点**: 検証段数を増やすほど精度↑・原価↑。KPI（bounce率・粗利）でチューニング。

---

## 8. 精度に関するKPI（[01](./01_事業計画.md)と連動）

| KPI | 目標イメージ |
|-----|-------------|
| メール bounce 率 | < 5%（検証済み分） |
| 再現実行の一致率 | > 95% |
| ICP適合の体感精度（ユーザー除外率） | 除外率 < 15% |
| 検証成功率（取得→有効連絡先） | 業界別にベースライン化し継続改善 |
