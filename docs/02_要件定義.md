# 02. 要件定義

## 1. スコープ全体像

本プロダクトは以下の7つの機能ドメインで構成する。

1. **認証・組織管理**（Auth / Workspace / Team）
2. **チャット & エージェント**（自然言語→検索プラン→実行）
3. **データソース検索**（コネクタ層でライブweb検索）
4. **検証・エンリッチメント**（多段検証＋信頼度スコア）
5. **リード管理**（テーブルUI・保存・エクスポート）
6. **クレジット & 課金**（消費計測・サブスク）※決済実装は最後
7. **公開API & CRM連携**（差別化ポイント）

各機能の詳細挙動は [03_機能仕様書](./03_機能仕様書.md) を参照。

## 2. 機能要件（FR）

### FR-1 認証・組織管理
- FR-1.1 メール＋OAuth（Google）でサインアップ/ログイン
- FR-1.2 ワークスペース（＝プロジェクト単位）の作成・切替
- FR-1.3 チームメンバー招待・ロール（Owner / Member）
- FR-1.4 プランに応じたワークスペース数・並列数の制限

### FR-2 チャット & エージェント
- FR-2.1 自然言語でICP（理想顧客像）を入力できる
- FR-2.2 エージェントが意図を解釈し「検索プラン」を提示（実行前に確認可）
- FR-2.3 実行状況をストリーミングで逐次表示（どのソースを検索中か）
- FR-2.4 会話履歴を保持し、追加指示で絞り込み・拡張ができる
- FR-2.5 **検索プランをスナップショット保存し、再実行で再現できる**（差別化）

### FR-3 データソース検索
- FR-3.1 コネクタ層で複数ソースを横断検索（地図/求人/企業サイト/EC検出/SNS 等）
- FR-3.2 ソースは市場（日本/グローバル）で切替可能
- FR-3.3 検索結果を正規化された「リード候補」に変換（名寄せ・重複排除）
- FR-3.4 各リードに「出典（どのソースで見つけたか）」を保持

### FR-4 検証・エンリッチメント
- FR-4.1 メール到達性の多段ウォーターフォール検証
- FR-4.2 電話番号の検証
- FR-4.3 エンリッチメント（技術スタック・広告出稿・トラフィック・企業規模等）
- FR-4.4 **各項目に信頼度スコア（0-100）を付与し可視化**（差別化）
- FR-4.5 検証成功（マッチ）時のみクレジット消費

### FR-5 リード管理
- FR-5.1 リードをテーブルで一覧・ソート・フィルタ
- FR-5.2 リストとして保存・命名・再利用
- FR-5.3 CSVエクスポート（無料）
- FR-5.4 リード単位で「除外/お気に入り」等のステータス管理

### FR-6 クレジット & 課金
- FR-6.1 クレジット残高の表示・消費履歴
- FR-6.2 消費が発生する操作の事前見積り提示
- FR-6.3 プラン別クレジット付与・月次リセット
- FR-6.4 Stripe連携（サブスク＋追加クレジット購入）※**最後に実装**

### FR-7 公開API & CRM連携
- FR-7.1 REST APIで検索・結果取得（APIキー発行）
- FR-7.2 HubSpot / Salesforce へリード書き出し
- FR-7.3 Webhook（ジョブ完了通知）
- FR-7.4 Sequencer（簡易な後続アクション用データ整形）

## 3. 非機能要件（NFR）

| ID | 分類 | 要件 |
|----|------|------|
| NFR-1 | 性能 | 検索ジョブは非同期。初回結果ストリーミングは目標30秒以内、全完了は数分許容 |
| NFR-2 | スケール | 検索は並列ワーカーで水平スケール。プラン別に同時実行数を制御 |
| NFR-3 | 信頼性 | ジョブは再試行可能・冪等。外部API失敗時は部分結果を返す |
| NFR-4 | セキュリティ | Supabase RLSでワークスペース分離。APIキーはハッシュ保存 |
| NFR-5 | プライバシー | B2B連絡先に限定、オプトアウト対応、保持期間管理、GDPR/個人情報保護法順守 |
| NFR-6 | 可観測性 | 構造化ログ、ジョブトレース、外部APIコスト計測、原価アラート |
| NFR-7 | コスト | 1クレジット原価 < 販売単価 を維持。キャッシュで重複API呼び出し削減 |
| NFR-8 | 保守性 | 外部依存はアダプタで抽象化し差し替え可能に |
| NFR-9 | 可用性 | MOCK_MODEで外部API未接続でも全機能デモ可能 |
| NFR-10 | 国際化 | 日本語/英語UI・多言語データ処理 |

---

## 4. 前提・制約

- 外部APIキー・決済は**最終フェーズで設定**。それまでモックアダプタで駆動。
- スクレイピングは各サイト規約・法令の範囲内。公式API優先。
- MVPはPLGを想定し、Freeプランで一通り体験できること。

---

## 5. スコープ外（初期リリースでは作らない）

- メール自動送信・シーケンス配信そのもの（データ整形までに留める）
- 高度なBIダッシュボード
- モバイルネイティブアプリ
- オンプレ提供
