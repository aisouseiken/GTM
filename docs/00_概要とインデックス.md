## 1. これは何か（1行）

**「理想の顧客像を自然言語で伝えるだけで、AIエージェントがリアルタイムでウェブを検索し、検証済みの営業リード（企業＋担当者の連絡先）を発掘するプラットフォーム」**

ベンチマーク: [origami.chat](https://origami.chat/) のクローン ＋ 精度・再現性の強化。

## 2. 既存サービス（origami.chat）との差別化方針

origami の弱点を潰すことを差別化の軸に置く。

| origami の弱点 | 本プロダクトの打ち手 | 参照 |
|----------------|---------------------|------|
| 同じプロンプトで結果がブレる（再現性が低い） | **検索プラン確定 → スナップショット保存 → 再実行で再現** | [08_精度向上戦略](./08_精度向上戦略.md) |
| 検証プロバイダが少ない（メール5/電話9） | **多段ウォーターフォール検証 ＋ 信頼度スコア** | [08_精度向上戦略](./08_精度向上戦略.md) |
| 公開API/CRM連携が弱い | **最初から公開REST API ＋ HubSpot/Salesforce書き出し** | [06_外部API連携設計](./06_外部API連携設計.md) |
| 実行が遅い（数分） | **ストリーミングで途中結果を逐次表示 ＋ ジョブ非同期化** | [04_システム設計書](./04_システム設計書.md) |
| 日本市場カバーが弱い（前提: 海外ソース中心） | **国内ソース（法人番号API/Indeed/地図等）をコネクタ化** | [06_外部API連携設計](./06_外部API連携設計.md) |

## 4. 技術スタック（確定）

- **フロント/バック統合**: Next.js 16 (App Router) + TypeScript + Tailwind CSS v4
- **エージェント層**: 現状はヒューリスティック（キーワード抽出）。本番はLLM接続に差し替え（最終フェーズ）
- **DB**: 現状は**インメモリ（開発用）**。**Prisma + PostgreSQL** のスキーマ/マイグレーションを `prisma/` に用意済み（`DATABASE_URL` で開発DBに接続）。将来 RLS/pgvector を検討
- **ジョブ**: サーバー内で非同期実行＋SSEで進捗配信（本番は専用ワーカーに拡張可）
- **認証**: 現状は**署名付きセッション（HMAC・モック）**。本格的なパスワード/OAuth認証は最終フェーズ（Supabase Auth 等を想定）
- **課金**: Stripe（Checkout / Portal / Webhook 実装済み）※ 鍵は最後に設定
- **ホスティング**: Vercel（アプリ）＋ 外部PostgreSQL（DB）

選定理由: **1言語（TS）で最速に組め、他人が読みやすく、採用市場も広い。**
