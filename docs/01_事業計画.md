# 01. 事業計画

## 1. ビジョン / ミッション

- **ビジョン**: 「営業リスト作りを、検索クエリの構築からAIへの一言に変える」
- **ミッション**: あらゆる営業チームが、鮮度100%・検証済みの見込み客リストを数分で手に入れられる状態をつくる。

---

## 2. 解く課題（Why now）

| 従来の営業リスト作成 | 課題 |
|--------------------|------|
| Apollo/ZoomInfo等の固定DBを検索 | データが古い（DB更新待ち）、フィルタ設定が煩雑 |
| オフライン業界（HVAC/歯科/飲食/建設等）| カバー率が低くリードが取れない |
| 手作業リサーチ | 1件あたり数十分、非スケール |

**Why now**: LLMの推論コスト低下＋ツール実行（エージェント）が実用化し、「リアルタイムweb検索でリスト生成」が現実的コストで可能になった。

---

## 3. ソリューションと提供価値

自然言語プロンプト → 自律エージェントがライブwebを検索 → 検証済みリードを納品。

**3つの価値**
1. **手軽さ**: フィルタ構築不要、プロンプト1つ
2. **鮮度**: DBではなく「今のweb」を検索
3. **品質**: 検証済み連絡先のみ課金（多段検証＋信頼度スコア）

---

## 4. ターゲット顧客（ICP）

| セグメント | 説明 | 優先度 |
|-----------|------|--------|
| スタートアップ/スケールアップのSDR/BDR | アウトバウンド営業チーム | ★★★（最優先） |
| 中小の代理店・受託（営業代行含む） | 顧客のためにリスト作成 | ★★ |
| ローカルサービス業（国内） | 地場のBtoB営業 | ★★（日本での差別化） |
| エンプラ営業組織 | 大量・API連携ニーズ | ★（Enterprise） |

---

## 5. 収益モデル

**サブスク ＋ クレジット従量制のハイブリッド**（origami踏襲＋改良）

### 課金対象（クレジット消費）
- 新規リード取得
- メール/電話の検証
- エンリッチメント（技術スタック・広告・トラフィック等）

### 常に無料
- エージェントとの会話・推論
- 自社データの変換・整形
- CSVエクスポート

### 料金プラン（暫定・origamiに準拠しつつ再現性機能で上位を厚く）

| プラン | 月額 | クレジット/月 | 主な差分 |
|--------|------|--------------|---------|
| Free | ¥0 / $0 | 1,000（1回のみ） | エクスポート不可 |
| Starter | ¥4,500 / $29 | 2,000 | 基本機能・CSV |
| Pro | ¥19,800 / $129 | 9,000 | 高精度モデル・再現実行・API |
| Scale | ¥74,800 / $499 | 40,000 | 高並列・優先実行 |
| Enterprise | 応相談 | 無制限/カスタム | SSO/SLA/専用ソース |

> クレジット単価と原価（外部API従量＋LLMトークン）の関係は [08](./08_精度向上戦略.md) と [06](./06_外部API連携設計.md) で管理。**粗利を守るため、1クレジット原価 < 販売単価 の維持を必達KPIとする。**

## 6. 競合と立ち位置

| 競合 | タイプ | 我々の差別化 |
|------|--------|-------------|
| origami.chat | リアルタイムAIエージェント | **再現性・検証幅・API・日本対応**で上回る |
| Apollo / ZoomInfo | 固定DB | 鮮度・オフライン業界・自然言語UX |
| Clay | データエンリッチのワークフロー | ノーコード不要、プロンプト起点の手軽さ |

**ポジショニング**: 「最も手軽（プロンプト1つ）× 最も信頼できる（多段検証＋再現性）リード発掘エージェント」

---

## 7. Go-To-Market

1. **PLG（プロダクト主導）**: Freeプラン→自己拡張。Product Hunt / X / SEO。
2. **業界特化コンテンツ**: 「HVACのリスト作り」「歯科クリニック開拓」等、オフライン業界のロングテールSEO。
3. **国内**: 営業代行・SaaS企業のSDRコミュニティ、法人番号連携を武器に。
4. **API/連携**: HubSpot/Salesforceマーケットプレイス掲載でリーチ拡大。

---

## 8. 主要KPI

| 区分 | KPI |
|------|-----|
| 獲得 | サインアップ数、Free→有料転換率 |
| 活用 | 週次アクティブ、平均リード生成数、検証成功率 |
| 品質 | 連絡先の到達率（bounce率）、再現実行の一致率 |
| 収益 | MRR、クレジット消費量、粗利率（販売単価−API原価） |
| 継続 | 月次チャーン、NRR |

---

## 9. リスクと対策

| リスク | 対策 |
|--------|------|
| 外部データソースの規約/法的リスク（スクレイピング等） | 公式API優先、規約順守、地域別コンプラ設計、データ削除対応 |
| 個人情報保護（GDPR / 個人情報保護法） | オプトアウト窓口、保持期間管理、B2B連絡先に限定 |
| API原価が販売単価を上回る | クレジット消費計測を厳格化、キャッシュ、原価アラート |
| 結果精度への不信 | 信頼度スコア可視化・出典明示・再現実行で信頼獲得 |
| LLM/データAPIのベンダーロック | アダプタ層で抽象化、差し替え可能に |
